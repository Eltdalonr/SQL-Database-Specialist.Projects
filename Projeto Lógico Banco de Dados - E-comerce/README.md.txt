

## **E-Commerce – Banco de Dados**

Este projeto apresenta a criação do modelo lógico, a inserção de dados e confecção de queries para responder perguntas estabelecidas, do modelo conceitual dele.
##

## **1. Descrição Geral**

O banco de dados foi criado para sedimentar os conhecimentos de modelagem conceitual, lógica e física de banco de dados:



* Replicação e o refinamento da modelagem lógica de um banco de dados para um cenário de e-commerce, a partir de um modelo conceitual
* Mapeamento de relacionamentos do modelo EER (Enhanced Entity-Relationship) para o modelo relacional.
* Desenvolvimento de script SQL completo para criação do esquema do banco de dados, seguido pela persistência de dados fictícios para fins de teste e validação.

* Foram elaboradas consultas SQL mais complexas, utilizando diferentes cláusulas, com o objetivo de explorar os dados e responder perguntas relevantes do contexto de negócio.

O projeto foi desenvolvido para fins educacionais e faz parte de um desafio prático voltado ao fortalecimento do portfólio em SQL e modelagem de dados, sendo disponibilizado em um repositório público no GitHub.

##

## **2. Objetivo**



1. Criar o script SQL de criação das tabelas;
2.Desenvolver queries SQL que utilizem:

SELECT

WHERE

Atributos derivados (expressões)

ORDER BY

HAVING

JOIN entre tabelas

3. Elaborar perguntas de negócio que possam ser respondidas pelas consultas


Quantos pedidos foram realizados por cada cliente?

Existe algum vendedor que também seja fornecedor?

Qual a relação entre produtos, fornecedores e estoques?

Quais são os nomes dos fornecedores e seus respectivos produtos?

Quais clientes possuem mais de uma forma de pagamento cadastrada?
